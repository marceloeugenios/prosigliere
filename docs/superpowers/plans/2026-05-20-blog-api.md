# Blog API Implementation Plan

**Goal:** Implement a RESTful blogging API (BlogPost + Comment) in this Spring Boot repo per SPEC.md, with full integration test coverage.

**Architecture:** Single Spring Boot app — JPA entities for BlogPost and Comment, Spring Data repositories, a service layer, and one REST controller mapped to `/api/v1/posts`. All four endpoints are protected by the existing `StaticTokenAuthFilter`. Integration tests use Testcontainers (MySQL + Redis) via the shared `BaseTest` fixture.

**Tech Stack:** Spring Boot 4.0, JPA/Hibernate, MySQL 8, Redis, Flyway, MapStruct, Testcontainers, Lombok, Bean Validation, SpringDoc OpenAPI, JUnit 5.

## Tasks

### Task 1: Domain entities and repositories
Create `BlogPost` and `Comment` JPA entities with UUID v7 IDs. Create `BlogPostRepository` with a `fetchById` LEFT JOIN FETCH query to avoid N+1, and `CommentRepository`.

### Task 2: DTOs, MapStruct mappers, service interface, and controller skeleton
Create request/response DTOs (records with Bean Validation). Create `BlogPostService` interface typed on entities. Create MapStruct mappers in `controller/mapper/`. Create `BlogPostServiceImpl` (all methods throw until implemented). Create `BlogPostController` skeleton.

### Task 3: TDD — POST /api/v1/posts
Write integration tests first, then implement `createPost`. Tests cover: 201 on valid request, 400 on blank title, 400 on missing content, 401 on missing token.

### Task 4: TDD — GET /api/v1/posts
Add integration tests, then implement `findAll` using `PagedResponse`. Tests cover: 200 with paginated results and commentCount, 401 on missing token.

### Task 5: TDD — GET /api/v1/posts/{id}
Add integration tests, then implement `findById` with `@Cacheable`. Tests cover: 200 with comments, 401, 404 with correct error message.

### Task 6: TDD — POST /api/v1/posts/{id}/comments
Add integration tests, then implement `saveComment` with `@CacheEvict` on the parent post. Tests cover: 201, 400, 401, 404. Include a cache eviction test that sequences: populate cache → write comment → re-fetch and assert the new comment appears.

### Task 7: Infrastructure — Flyway, Redis cache, security
Replace `ddl-auto: create-drop` with Flyway migrations. Wire `FlywayConfig` manually (Spring Boot 4.0). Configure `CacheConfig` with a 10-minute TTL Redis cache manager. Externalize the auth token via `API_BLOG_TOKEN`. Add request body size limits.

### Task 8: Unit tests — BlogPostServiceImpl
Add `BlogPostServiceImplTest` with no Spring context. Cover: createPost, findAll sort/pagination params, findById happy path, findById not found, saveComment associations, saveComment not found.

### Task 9: API versioning and OpenAPI annotations
Version all endpoints under `/api/v1/`. Add OpenAPI annotations to the controller and request DTOs.

### Task 10: Evaluator experience
Create `docker/docker-compose-cloud.yml` and `docker/.env.example`. Update README with evaluator quickstart requiring only Docker.
